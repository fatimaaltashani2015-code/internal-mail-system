# نظام إدارة البريد الداخلي

موقع إلكتروني لإدارة البريد الداخلي للإدارة عن طريق قسم البريد والأقسام المختصة.

## المتطلبات

- Node.js 18 أو أحدث
- npm أو yarn

## التثبيت والتشغيل

### 1. تثبيت Node.js (إن لم يكن مثبتاً)

حمّل Node.js من [nodejs.org](https://nodejs.org/) وثبّته على جهازك.

### 2. تثبيت المشروع

```bash
cd internal-mail-system
npm install
```

### 3. إعداد ملف البيئة

انسخ ملف البيئة (إن لم يكن موجوداً):

```bash
copy .env.example .env
```

### 4. إعداد قاعدة البيانات

**يُستخدم PostgreSQL.** راجع [POSTGRES_SETUP.md](POSTGRES_SETUP.md) للتفاصيل.

```bash
npx prisma generate
npx prisma migrate deploy
npx prisma db seed
```

### 5. تشغيل المشروع

```bash
npm run dev
```

افتح المتصفح على: [http://localhost:3000](http://localhost:3000)

## بيانات الدخول الافتراضية

بعد تشغيل الأمر `db:seed`:

| الصلاحية   | الرقم الوظيفي | كلمة المرور |
|-----------|---------------|-------------|
| مدير النظام | 0001          | admin123    |
| قسم البريد  | 1001          | admin123    |
| قسم تقنية المعلومات | 2001 | admin123    |
| قسم الموارد البشرية | 2002 | admin123    |

## البيانات التجريبية

يضيف الأمر `db:seed` بيانات تجريبية تشمل:
- **20 رسالة** موزعة على حالات مختلفة (لم يتم الاطلاع، تم الاطلاع لم يتم الرد، تم الرد)
- **ملفات مرفقة** لكل رسالة (PNG, PDF, JPG)
- رسائل قديمة (أكثر من 3 أيام) تظهر بالإشارة الحمراء للتنبيه

## هيكل الواجهات

### واجهة تسجيل الدخول
- إدخال الرقم الوظيفي وكلمة المرور
- عرض اسم المستخدم والصفة في أعلى الواجهة بعد الدخول

### موظفي قسم البريد
1. **متابعة البريد** - 3 جداول (لم يتم الاطلاع، تم الاطلاع لم يتم الرد، تم الرد)
2. **بريد الصادر** - إدخال رسائل جديدة
3. **البريد الوارد** - عرض الرسائل المحالة
4. **فتح رسالة** - عرض التفاصيل والرد

### موظفي الأقسام الأخرى
1. **البريد الوارد** - رسائل محالة للقسم
2. **فتح رسالة** - عرض والرد

### مدير النظام
1. **إدارة الأقسام** - إضافة، بحث، تعديل
2. **إدارة المستخدمين** - إضافة، بحث، تعديل

## التقنيات المستخدمة

- **Tailwind CSS** - التنسيق

- **PostgreSQL** - قاعدة البيانات
